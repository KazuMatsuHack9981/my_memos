#docker images
	現在持っているDockerイメージ一覧を取得
	
#docker pull
	dockerイメージを入手。入手したものはdocker imagesで見れる。
	
#dockerの立ち上げ
	sudo systemctl start docker : Dockerを立ち上げる
	sudo systemctl stop docker :  Dockerを停止
	※sudo docker images の表示で今どっちなのかわかる
	
#docker ps / docker-compose ps
	docker runしたコンテナの状況が見れる.
	-a をつけると停止しているコンテナも表示される。
	
	
#docker stop コンテナID
	これでdocker runしたコンテナを終了できる
	
#docker-compose up
	これでコンテナを開始できる。
	-d オプションをつければバックグランドで起動でき、こちらのほうが便利。
	
#docker-compose stop
	これでコンテナを停止できる。
	-d がないときは、Ctl+Cした時点でStopになっている。
	
#docker rmi IMAGEID -f
	これでそのIDのイメージを強制削除。たとえばnoneとかのイメージ。
	
	
#dockerstats
	自分で設定した関数。docker images, docker ps, docker-compose psを全て行う。※sudo付きで書いてある。
	
	
	
#resetdocker
	コンテナ全削除 かつ イメージ全削除　(危ないので間違えて打たないように!)
	
#sudo docker rm -f $(sudo docker ps -a -q)
	これでコンテナが全削除される。
	
#sudo docker images -aq | xargs sudo docker rmi
	これでイメージが全削除される。
	
	
	
	
	
#docker exec -it コンテナ名 bash
	コンテナの中に入る。exitででることができる。
